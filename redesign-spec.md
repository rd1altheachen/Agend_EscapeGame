# 改版規格：美術場景化 + 探索式互動

## 核心改動原則

1. **場景視覺化**：用 CSS 繪製詳細的房間場景（漸層、形狀、陰影），讓玩家感覺身處真實的魔法房間
2. **隱藏熱點探索**：互動物件不標示文字/icon，玩家需要自己探索發現（hover 時 cursor 變手指 + 微微發光即可）
3. **移除過度提示**：謎題 modal 中不直接顯示答案或過於明確的提示
4. **物品欄修復**：確保物品欄始終可見且正確顯示收集的道具

## 二、互動方式改版

### 熱點設計原則：
- 熱點區域是透明的 `div`，覆蓋在場景物件上方
- **預設狀態**：完全不可見（no border, no background, no text）
- **Hover 狀態**：cursor 變為 pointer + 物件微微亮起（opacity 或 brightness 變化）
- **不顯示任何文字標籤**
- 玩家需要自己移動滑鼠探索哪裡可以點擊

### 手機適配：
- 手機沒有 hover，所以可互動區域在長按時顯示微弱光暈
- 或者：場景中偶爾有微弱的粒子效果飄過可互動物件（暗示這裡有東西）

## 三、謎題 UI 改版（移除過度提示）

### 原則：
- Modal 中只描述「玩家看到什麼」，不直接告訴玩家該怎麼做
- 不在密碼鎖旁邊寫提示文字
- 線索分散在環境中，玩家需要自己連結

### 具體修改：

**謎題 1-1 魔法鏡：**
- 原本：直接說「第二排中間有一本書在發光」
- 改為：鏡中場景與現實有微妙差異，玩家需要自己比對發現哪本書不同

**謎題 1-2 日記符號鎖：**
- 原本：直接列出月相順序和對應符號
- 改為：日記每頁只顯示插圖和邊角符號，玩家需要自己推理月相順序

**謎題 1-5 壁爐密碼：**
- 原本：密碼鎖下方直接寫「紅?、藍?、綠?、紫?」
- 改為：只顯示密碼鎖，沒有任何提示文字

**謎題 2-1 音樂盒：**
- 原本：直接寫「節奏：長-短-短-長」
- 改為：播放旋律時用視覺動畫表示節奏（按鈕亮的時間長短不同），玩家需要自己觀察

**謎題 2-4 櫥櫃：**
- 原本：密碼鎖下方寫「水晶球裡看到掛鐘3:15」
- 改為：只顯示密碼鎖，線索需要玩家自己從水晶球和掛鐘連結

**所有密碼鎖/輸入介面：**
- 只顯示輸入框和確認按鈕
- 不在旁邊寫任何提示文字
- 錯誤時只顯示「不對...」不給額外提示

## 四、物品欄修復

- 確保 `#inventory-bar` 始終可見（即使沒有物品也顯示空的欄位）
- 物品欄背景加深，與場景有明確分隔
- 空物品欄顯示淡色文字「物品欄」或虛線框暗示這裡會放東西
- 收集物品時有明確的動畫（物品飛入物品欄）

---

## 五、提示系統調整

- 💡 提示按鈕保留，但提示內容改為更模糊的引導
- 第1層：暗示方向（「這個房間裡有面鏡子...」）
- 第2層：稍微具體（「鏡中的世界和現實有什麼不同？」）
- 第3層：明確但不直接給答案（「仔細比對鏡中書架和現實書架的差異」）

---

## 六、技術實作要點

- 場景用純 CSS 繪製（div + gradient + border-radius + box-shadow）
- 熱點用 absolute positioned 透明 div
- 場景需要響應式（用 % 定位 + vw/vh 尺寸）
- 保持現有的遊戲邏輯不變，只改 UI 層
